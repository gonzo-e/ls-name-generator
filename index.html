Installing the Plugin

Two ways to install — pick whichever fits your team's setup. Grab the files from the Download box near the top of this page first.

⭐ Recommended

Private Organization Plugin

If your team is on a Figma Organization or Enterprise plan, you can publish this as a private plugin. Go to Admin → Plugins → Publish private plugin, upload the three files, and every team member can install it with one click from the Figma plugin browser — no local files needed, updates deploy instantly.

Dev Mode

Local Development Install

Works on any Figma plan. Download the plugin folder (all three files: code.js, ui.html, manifest.json) and keep them together. In Figma go to Main Menu → Plugins → Development → Import plugin from manifest… and select manifest.json. Run it anytime via ⌘ / and search LS Name Generator. Each team member needs a copy of the files.

How to Use It

Three steps, every time.

1

Select your layers in Figma

Click any frame, component, slice, or group you want to rename or export. You can select multiple layers at once using Shift+click or ⌘+click. The plugin will process all of them.

2

Fill in the fields

Open the plugin and enter your Campaign Name and Trello ID. The Live Preview updates as you type so you can see the exact output filename before committing. Choose your export format (PNG, JPG, SVG, PDF) and scale if you're exporting.

3

Rename or Export

Hit Rename Layers to update the layer names in Figma without downloading anything. Hit ⬇ Rename & Export to rename and download the files. Each selected layer downloads as its own individual file — no ZIPs.

Field Reference

What each control does.

Campaign Name
Required. The name of the campaign or project. Spaces are automatically converted to underscores. Special characters are stripped. Example: Fall Promo → fall_promo
Trello ID
The card ID from your Trello ticket. Optional but recommended for traceability. Casing is always preserved exactly as entered — Trello short links are case-sensitive, so dbTQbJ1C stays dbTQbJ1C and never gets lowercased like the rest of the filename.
Suffix / Variant
Appends a version tag to the name. Options: _v2, _v3, _final, _approved, _revised. Leave as None if not needed.
Add _SKU_
When checked, the plugin automatically detects the SKU number from the layer's Figma annotation and appends it as _SKU_379254. If no annotation is found it falls back to _SKU_000000 rather than a bare placeholder, so the filename format stays consistent. See the SKU Auto-Detection section below for full details.
Increment
When exporting multiple layers, adds _01, _02, _03… to each file so they're individually identifiable. Only applies when 2+ layers are selected.
Format
Export file type: PNG (default, supports transparency), JPG (smaller, no transparency), SVG (vector), PDF (print/presentation).
Scale
Resolution multiplier for PNG and JPG. 1x = screen resolution. 2x = retina / standard web. 3x = high-res / mobile. Not applicable for SVG or PDF.
📋 Copy Name
Copies the previewed filename to your clipboard without renaming or exporting anything. Useful for referencing in Slack, Notion, or Trello.
✕ Clear
Resets all fields and clears the saved session. Use this when switching to a new campaign.
SKU Auto-Detection

How the plugin finds and attaches SKU numbers automatically.

When the Add _SKU_ checkbox is checked, the plugin doesn't just append a generic placeholder — it actively reads the Figma annotation attached to each layer and extracts the SKU number from it, so every exported file is named with the real product identifier.

1

Add a Figma annotation to your frame

In Figma, select a frame and open the Annotations panel (via the right sidebar or Shift+A). Add an annotation and type the SKU number as the label — e.g. 379254. That blue numbered badge you see floating to the right of the frame is the annotation.

2

The plugin scans each selected layer

When you hit Rename Layers or Rename & Export, the plugin reads node.annotations directly on each selected layer. It uses a pattern match looking for any 5–7 digit number in the annotation label — that's the expected format for SKU numbers.

3

Fallback: child text layers

If no annotation is found on the frame itself, the plugin recurses into its child layers looking for a text layer whose entire content is just a 5–7 digit number. This covers cases where SKUs are placed as text inside the frame rather than as a Figma annotation.

4

The number is embedded in the filename

Once found, the SKU is appended as _SKU_379254. If no number is detected anywhere, it falls back to _SKU_000000 so the filename format stays consistent and downstream tools parsing for a SKU segment don't break on a missing value.

Without annotation
260520_fall_promo_dbTQbJ1C_SKU_000000
With annotation (SKU detected)
260520_fall_promo_dbTQbJ1C_SKU_379254
Important: The live preview in the plugin always shows _SKU_000000 as a placeholder — the actual number is only resolved at rename/export time when the plugin can read the selected layer's annotation from Figma.
Tips & Gotchas
💾

It remembers you

The plugin saves your last-used campaign, Trello ID, format, and settings. They auto-fill next time you open it.

🗂

Multi-layer exports

Select 2+ layers and hit Rename & Export — each layer downloads as its own individual file, one after the other.

⚠️

Not all layers export

Pages and some connector types can't be exported. The plugin skips them gracefully and tells you how many were skipped.

🔡

Everything is lowercase

The output name is always lowercased automatically. No need to worry about casing when you type — it's handled.

🧹

Special chars are stripped

Characters like / : * ? " < > # etc. are removed automatically so filenames are always OS-safe.

📏

100-char limit

Names are capped at 100 characters. The live preview turns red when you're getting close so you can shorten before exporting.
